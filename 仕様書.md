# 資格学習アプリ 仕様書

## 1. 概要・目的

資格取得を趣味とする人向けの、学習を一元管理する Web アプリ。
複数の資格を「試験日までのカウントダウン」とともに管理し、暗記カードによる復習、勉強時間の記録、参考書の進捗管理、タスク管理までを 1 つのアプリで完結させる。

**なぜ作るか（ポートフォリオでの語り）**
自分自身が資格勉強をする中で「勉強時間・進捗・暗記がバラバラのツールに散っていて不便」と感じたことが出発点。実体験の課題を、自分で設計して解決した作品。

## 2. ターゲットユーザー

- 複数の資格を並行して、あるいは次々に受験する人
- 勉強時間や進捗を記録してモチベーションを保ちたい人

## 3. 設計の中心となる考え方

すべてのデータは「資格（Certification）」を親（ハブ）として、そこに 4 種類の子要素がぶら下がる **ハブ＆スポーク構造**。
資格を 1 件削除すると、それにひもづくカード・Todo・学習記録・参考書もまとめて消える（一対多の関係）。

```
                ┌─ Flashcard（暗記カード）
                ├─ Todo（タスク）
Certification ──┤
                ├─ StudySession（学習記録）
                └─ Textbook（参考書）
```

## 4. 機能一覧と優先順位

開発は「1 機能ずつ画面も API も完成させてから次へ」という**縦スライス方式**で進める。
どこで止まっても “動く作品” が手元に残るのが狙い。

| 優先 | 機能 | 内容 | 区分 |
|------|------|------|------|
| 1 | 資格管理 | 資格の登録・一覧・編集・削除＋試験日カウントダウン | MVP・土台 |
| 2 | 学習タイマー | タイマーで計測し、勉強時間を記録。資格ごと・合計の総時間を表示 | MVP |
| 3 | 参考書進捗 | 参考書を登録し「今どこの章まで進んだか」を記録・表示 | MVP |
| 4 | 暗記カード | カードを登録し、間隔反復（Leitner 方式）で復習 | MVP・技術的見せ場 |
| 5 | Todo | 資格ごとのタスクを追加・完了・削除 | MVP |
| - | 分野別正答率グラフ | カードや学習の統計を可視化 | 発展 |
| - | 本格的な間隔反復（SM-2） | 正答率に応じて間隔を細かく最適化 | 発展 |
| - | 更新期限リマインド | 更新が必要な資格の期限通知 | 発展 |
| - | 取得済みコレクション表示 | 取得済み資格を実績として一覧・バッジ表示 | 発展 |

## 5. データモデル

### Certification（資格）— 親
| カラム | 型 | 説明 |
|--------|----|----|
| id | int (PK) | 主キー |
| name | str | 資格名 |
| exam_date | date / null | 試験日（カウントダウンに使用。未定なら空） |
| status | str | 状態（studying = 勉強中 / acquired = 取得済み） |

### Flashcard（暗記カード）
| カラム | 型 | 説明 |
|--------|----|----|
| id | int (PK) | 主キー |
| certification_id | int (FK) | どの資格のカードか |
| question | str | 問題（カードの表） |
| answer | str | 答え（カードの裏） |
| box | int | Leitner の箱番号（1〜5）。初期値 1 |
| next_due | date / null | 次回出題日 |

### Todo（タスク）
| カラム | 型 | 説明 |
|--------|----|----|
| id | int (PK) | 主キー |
| certification_id | int (FK) | どの資格のタスクか |
| title | str | 内容 |
| is_done | bool | 完了フラグ |

### StudySession（学習記録）
タイマーで 1 回勉強するごとに 1 行追加される。総勉強時間はこの記録を合計して算出する（タイマーの状態そのものは保存しない）。
| カラム | 型 | 説明 |
|--------|----|----|
| id | int (PK) | 主キー |
| certification_id | int (FK) | どの資格の勉強か |
| duration_min | int | 学習時間（分） |
| studied_at | date | 学習日 |

### Textbook（参考書）
| カラム | 型 | 説明 |
|--------|----|----|
| id | int (PK) | 主キー |
| certification_id | int (FK) | どの資格の参考書か |
| title | str | 書名 |
| current_chapter | int | 今どこまで進んだか（現在の章）。初期値 0 |
| total_chapters | int / null | 全章数（任意） |

## 6. API エンドポイント一覧

REST の基本（作成 = POST / 取得 = GET / 更新 = PATCH / 削除 = DELETE）に沿う。

### 資格（スライス 1）
| メソッド | パス | 内容 |
|----------|------|------|
| POST | /certifications | 資格を登録 |
| GET | /certifications | 資格一覧（各件に残り日数を含む） |
| GET | /certifications/{id} | 資格 1 件の詳細 |
| PATCH | /certifications/{id} | 資格を更新 |
| DELETE | /certifications/{id} | 資格を削除 |

### 学習記録（スライス 2）
| メソッド | パス | 内容 |
|----------|------|------|
| POST | /certifications/{id}/sessions | 勉強記録を 1 件保存 |
| GET | /certifications/{id}/sessions | 記録一覧＋合計時間 |

### 参考書（スライス 3）
| メソッド | パス | 内容 |
|----------|------|------|
| POST | /certifications/{id}/textbooks | 参考書を登録 |
| GET | /certifications/{id}/textbooks | 参考書一覧 |
| PATCH | /textbooks/{id} | 進捗（現在の章）などを更新 |
| DELETE | /textbooks/{id} | 参考書を削除 |

### 暗記カード（スライス 4）
| メソッド | パス | 内容 |
|----------|------|------|
| POST | /certifications/{id}/cards | カードを登録 |
| GET | /certifications/{id}/cards | カード一覧 |
| GET | /certifications/{id}/cards/due | 今日復習すべきカードだけ取得 |
| POST | /cards/{id}/review | 復習結果（正解/不正解）を送り、箱と次回出題日を更新 |
| PATCH | /cards/{id} | カードを編集 |
| DELETE | /cards/{id} | カードを削除 |

### Todo（スライス 5）
| メソッド | パス | 内容 |
|----------|------|------|
| POST | /certifications/{id}/todos | タスクを追加 |
| GET | /certifications/{id}/todos | タスク一覧 |
| PATCH | /todos/{id} | 完了切り替え・編集 |
| DELETE | /todos/{id} | タスク削除 |

## 7. 間隔反復（Leitner システム）のロジック

カードを「箱 1〜5」で管理し、正解で 1 つ上の箱へ、不正解で箱 1 へ戻す。
箱の番号が大きいほど、次に出題されるまでの間隔が長くなる。

| 箱 | 次回出題までの間隔（例） |
|----|--------------------------|
| 1 | 翌日 |
| 2 | 3 日後 |
| 3 | 1 週間後 |
| 4 | 2 週間後 |
| 5 | 1 ヶ月後 |

**復習時の処理（POST /cards/{id}/review）**
1. 正解なら `box` を +1（最大 5）、不正解なら `box` を 1 に戻す。
2. 更新後の `box` に対応する間隔で `next_due`（次回出題日）= 今日 + 間隔 を計算して保存。

**今日のカード抽出（GET /cards/due）**
`next_due <= 今日` のカードだけを返す。

> 発展形：将来は SM-2 アルゴリズム（正答の質を 0〜5 で評価し、間隔係数を動的に調整）に差し替え可能。まずは上記の決め打ち間隔で十分に機能する。

## 8. 技術スタック

| 区分 | 採用技術 | 選定理由 |
|------|----------|----------|
| 言語 | Python | 得意分野 |
| Web フレームワーク | FastAPI | 学習コストが低く、API ドキュメント（Swagger UI）が自動生成される |
| DB 操作（ORM） | SQLModel | SQLAlchemy と Pydantic を一体化した、FastAPI 作者製のライブラリ。初心者でも書きやすい |
| データベース | SQLite | ファイル 1 つで動き、環境構築が不要。開発・ポートフォリオに最適 |
| サーバー | Uvicorn | FastAPI の標準的な実行サーバー |
| フロント | （後半で）HTML/CSS/JavaScript | まずは API を `/docs` で動作確認し、フロントは後で追加する |

> SQLite は将来 PostgreSQL などに差し替え可能。まずは設定不要な SQLite で開発を進める。

## 9. 開発ロードマップ

### 全体の進め方
1 機能（スライス）ごとに「モデル定義 → API 実装 → /docs で動作確認」を 1 セットとして完成させ、次のスライスへ進む。

### ステップ
1. **環境構築** … Python 仮想環境を作り、ライブラリを入れ、サーバーを起動して `/docs` が開けることを確認する。
2. **スライス 1：資格管理** … 資格の CRUD ＋カウントダウンを実装（本リポジトリで実装済み）。
3. **スライス 2：学習タイマー** … 学習記録の保存・合計表示を実装。タイマー画面（フロント）はストップ時に記録を保存する形にする。
4. **スライス 3：参考書進捗** … 参考書の登録と「現在の章」の更新を実装。
5. **スライス 4：暗記カード** … カードの登録と Leitner 方式の復習ロジックを実装（このアプリの技術的な核）。
6. **スライス 5：Todo** … タスクの追加・完了・削除を実装。
7. **フロントエンド整備** … 各機能の画面を作り、見た目を整える。
8. **発展機能** … 統計グラフ、SM-2、リマインドなどを時間に応じて追加。
